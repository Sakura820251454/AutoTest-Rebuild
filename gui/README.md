# gui - 图形界面模块

AutoTest 图形用户界面模块，基于 PyQt5 实现，提供直观的配置管理、硬件检测和测试执行界面。

## 文件清单

### 主模块

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `__init__.py` | 辅助 | 模块初始化 |
| `main_window.py` | 核心 | GUI主窗口，整合所有面板和菜单 |

### 界面组件 (widgets/)

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `config_panel.py` | 核心 | 配置面板，编辑和显示配置信息 |
| `hardware_panel.py` | 核心 | 硬件检测面板，检测XDS100连接状态 |
| `execute_panel.py` | 核心 | 执行面板，控制测试执行和显示进度 |
| `log_panel.py` | 辅助 | 日志面板，显示执行日志 |
| `case_table.py` | 辅助 | 用例表格，显示用例执行状态 |
| `path_selector.py` | 工具 | 路径选择器组件，文件/目录选择 |
| `status_indicator.py` | 工具 | 状态指示器组件，显示连接状态 |

### 工作线程 (workers/)

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `pipeline_worker.py` | 核心 | 流水线工作线程，后台执行测试 |
| `hardware_checker.py` | 辅助 | 硬件检测线程，后台检测硬件连接 |

### 对话框 (dialogs/)

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `about_dialog.py` | 辅助 | 关于对话框 |
| `error_dialog.py` | 辅助 | 错误对话框 |
| `hardware_error_dialog.py` | 核心 | 硬件错误对话框，断点续测提示 |

### 工具函数 (utils/)

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `validators.py` | 工具 | 输入验证函数 |

## 主要功能

### 1. 配置管理
- 可视化编辑所有配置项
- 路径选择器支持文件/目录浏览
- 配置验证和保存
- 支持新旧配置格式

### 2. 硬件检测
- 快速检测XDS100调试器连接
- USB设备预检测（避免DSS超时）
- 详细的连接状态显示
- 自动识别设备信息

### 3. 测试执行
- 阶段自动检测（生成/构建/测试）
- 灵活的起始阶段选择
- 实时进度显示
- 用例状态实时更新（等待/运行/成功/失败）
- 断点续测支持

### 4. 硬件错误处理
- 自动检测硬件连接中断
- 弹出硬件错误对话框
- 支持断点续测恢复
- 自动重连提示

## 信号机制

### MainWindow 信号
- `config_loaded`: 配置加载完成
- `stage_detected`: 阶段检测完成

### ConfigPanel 信号
- `config_changed`: 配置变更

### HardwarePanel 信号
- `check_completed`: 硬件检测完成

### ExecutePanel 信号
- `execution_started`: 执行开始
- `execution_finished`: 执行完成
- `log_message`: 日志消息

## 工作线程

### PipelineWorker
后台执行流水线任务，避免阻塞UI：
- 进度更新信号
- 用例开始/完成信号
- 硬件错误信号
- 日志消息信号

### HardwareChecker
后台检测硬件连接：
- 检测完成信号
- 日志消息信号

## 使用方式

```bash
# 启动GUI
python run_gui.py

# 启动并加载配置
python run_gui.py -c config/config.json

# 自动安装依赖
python run_gui.py --install-deps
```
