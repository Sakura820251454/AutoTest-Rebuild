# 评估指标体系

本文档定义了进化式代码优化的评估标准和评分方法。

---

## 评估维度

优化质量由三个核心维度综合评估：

```
综合评分 = f(性能, 正确性, 可维护性)
```

---

## 1. 性能指标 (Performance Metrics)

### 1.1 响应时间 (Response Time)

**测量方法**：
- 单次操作延迟
- 分位数统计：P50, P90, P99

**评分标准**：

| 等级 | P99延迟 | 得分 |
|------|---------|------|
| 优秀 | < 100ms | 1.0 |
| 良好 | < 500ms | 0.8 |
| 合格 | < 1000ms | 0.6 |
| 需优化 | < 3000ms | 0.4 |
| 不合格 | >= 3000ms | 0.2 |

### 1.2 吞吐量 (Throughput)

**测量方法**：
- 单位时间内处理请求数
- TPS (Transactions Per Second)
- QPS (Queries Per Second)

**评分标准**：

| 等级 | 提升比例 | 得分 |
|------|----------|------|
| 优秀 | > 50% | 1.0 |
| 良好 | > 20% | 0.8 |
| 合格 | > 10% | 0.6 |
| 轻微 | > 5% | 0.4 |
| 无效 | <= 5% | 0.2 |

### 1.3 资源效率

**测量指标**：
- CPU利用率
- 内存使用效率
- I/O效率
- 网络带宽利用率

**评分标准**：

| 等级 | 资源利用率 | 得分 |
|------|------------|------|
| 优秀 | > 80% | 1.0 |
| 良好 | > 60% | 0.8 |
| 合格 | > 40% | 0.6 |
| 需优化 | > 20% | 0.4 |
| 不合格 | <= 20% | 0.2 |

---

## 2. 正确性指标 (Correctness Metrics)

### 2.1 功能正确性

**测量方法**：
- 测试用例通过率
- 边界条件处理
- 错误处理完整性

**评分标准**：

| 等级 | 测试通过率 | 得分 |
|------|------------|------|
| 通过 | 100% | 1.0 |
| 部分通过 | >= 95% | 0.5 |
| 失败 | < 95% | 0 |

**重要**：正确性得分 < 1.0 时，综合评分直接为 0

### 2.2 稳定性

**测量指标**：
- 多次执行结果一致性
- 无内存泄漏
- 无资源泄漏
- 无竞态条件

**评分标准**：

| 等级 | 表现 | 得分 |
|------|------|------|
| 稳定 | 无任何问题 | 1.0 |
| 轻微不稳定 | 偶发问题 | 0.7 |
| 不稳定 | 频发问题 | 0.3 |
| 严重不稳定 | 必定失败 | 0 |

---

## 3. 可维护性指标 (Maintainability Metrics)

### 3.1 代码复杂度

**测量方法**：
- 圈复杂度 (Cyclomatic Complexity)
- 认知复杂度 (Cognitive Complexity)
- 嵌套深度

**评分标准**：

| 等级 | 圈复杂度 | 得分 |
|------|----------|------|
| 优秀 | < 10 | 1.0 |
| 良好 | < 20 | 0.8 |
| 合格 | < 30 | 0.6 |
| 需重构 | < 50 | 0.4 |
| 不合格 | >= 50 | 0.2 |

### 3.2 代码可读性

**测量指标**：
- 命名清晰度
- 注释完整性
- 代码格式一致性
- 函数长度

**评分标准**：

| 等级 | 表现 | 得分 |
|------|------|------|
| 优秀 | 清晰易懂 | 1.0 |
| 良好 | 基本清晰 | 0.8 |
| 合格 | 可理解 | 0.6 |
| 需改进 | 难以理解 | 0.4 |
| 不合格 | 晦涩难懂 | 0.2 |

### 3.3 耦合度

**测量方法**：
- 模块间依赖关系
- 类间耦合度
- 函数间耦合度

**评分标准**：

| 等级 | 耦合程度 | 得分 |
|------|----------|------|
| 优秀 | 低耦合 | 1.0 |
| 良好 | 中等耦合 | 0.8 |
| 合格 | 可接受 | 0.6 |
| 需改进 | 高耦合 | 0.4 |
| 不合格 | 紧耦合 | 0.2 |

## 4. 综合评分函数

### 评分公式

```python
def fitness_score(performance, correctness, maintainability):
    """
    综合适应度评分
    
    Args:
        performance: 性能得分 (0-1)
        correctness: 正确性得分 (0-1)
        maintainability: 可维护性得分 (0-1)
    
    Returns:
        综合评分 (0-1)
    """
    if correctness < 1.0:
        return 0
    
    weights = {
        'performance': 0.5,
        'correctness': 0.2,
        'maintainability': 0.3
    }
    
    return (
        weights['performance'] * performance +
        weights['correctness'] * correctness +
        weights['maintainability'] * maintainability
    )
```

### 权重说明

| 维度 | 权重 | 说明 |
|------|------|------|
| 性能 | 50% | 优化的核心目标 |
| 可维护性 | 30% | 确保长期可维护 |
| 正确性 | 20% | 基础保障（一票否决） |

## 5. 评估流程

### 5.1 评估流水线

```
┌─────────────────────────────────────────────────────────────┐
│                      评估流水线                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 语法检查                                                 │
│     - 编译通过                                               │
│     - 无语法错误                                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 类型检查 (如有类型系统)                                   │
│     - 类型正确                                               │
│     - 无类型错误                                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 功能测试                                                 │
│     - 单元测试                                               │
│     - 集成测试                                               │
│     - 边界测试                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 性能测试                                                 │
│     - 基准测试                                               │
│     - 压力测试                                               │
│     - 对比测试                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  5. 复杂度分析                                               │
│     - 圈复杂度                                               │
│     - 代码行数                                               │
│     - 依赖分析                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  6. 计算综合评分                                             │
│     - 汇总各维度得分                                         │
│     - 应用权重计算                                           │
│     - 生成评估报告                                           │
└─────────────────────────────────────────────────────────────┘
```

## 6. 验收标准

### 最低验收标准

| 指标 | 最低要求 |
|------|----------|
| 正确性 | 100% 测试通过 |
| 性能提升 | >= 5% |
| 圈复杂度 | 不增加超过 20% |

### 优秀标准

| 指标 | 优秀要求 |
|------|----------|
| 正确性 | 100% 测试通过 |
| 性能提升 | >= 20% |
| 圈复杂度 | 降低或不变 |
| 综合评分 | >= 0.8 |
