# src - 核心代码模块

AutoTest 核心功能模块，提供配置管理、工程生成、构建、测试执行等核心能力。

---
一旦我所属的文件夹有所变化请更新我

## 文件清单

| 文件名 | 地位 | 功能 |
|--------|------|------|
| `__init__.py` | 辅助 | 模块初始化，导出主要类 |
| `config.py` | 核心 | 配置管理，支持新旧格式兼容、环境变量、路径解析 |
| `logger.py` | 核心 | 日志系统，彩色控制台输出、文件日志、上下文管理 |
| `exceptions.py` | 核心 | 自定义异常类，按错误码分类（1xxx-9xxx） |
| `generator.py` | 核心 | 工程生成，支持模板模式和手动模式 |
| `builder.py` | 核心 | 工程构建，并行多线程构建CCS工程 |
| `executor.py` | 核心 | 测试执行，DSS脚本调用、结果收集、断点续测 |
| `pipeline.py` | 核心 | 流水线管理，串联所有步骤、断点续传 |
| `hardware_detector.py` | 辅助 | 硬件预检测，USB设备检测、快速连接检查 |

## 主要类说明

### Config (config.py)
配置管理类，支持：
- 新旧两种配置格式自动兼容
- 环境变量替换（`${VAR}` 或 `%VAR%`）
- 相对路径解析（相对于配置文件目录）
- 参数验证和默认值

### ProjectGenerator (generator.py)
工程生成器，支持两种模式：
- **template 模式**: 从模板工程 + 源文件批量生成
- **manual 模式**: 验证已手动配置的工程

### ProjectBuilder (builder.py)
工程构建器，特性：
- 多线程并行构建
- 自动导入工程到CCS工作空间
- 构建产物自动收集

### TestExecutor (executor.py)
测试执行器，特性：
- DSS脚本自动生成和执行
- 内存数据导出
- 批次执行和断点续测
- 硬件连接检测

### HardwareDetector (hardware_detector.py)
硬件预检测器，功能：
- USB设备快速检测（Windows/Linux）
- XDS100调试器识别
- 避免DSS超时等待
